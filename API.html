API简介




什么是API

API（应用程序编程接口）
是一些预先定义的函数
目的：提供 应用程序与开发人员 基于某软件或硬件 得以访问一组例程的 能力
而又无需 访问源码或理解内部工作机制的细节





API的特点

是一个明确定义的接口， 可以 为其他软件 提供特定的服务
可以小到只包含一个单独的函数，也可以大到包含N多的类、方法、全局函数、数据类型、枚举类型和常量等
既可以是私有的，也可以是开源的




API的分类

面向对象语言的API；例如：Java API列表
库与框架的API；例如：Windows API、Windows DirectX
API与协议；例如：LDAP应用程序接口
API与设备接口；例如：PC BIOS调用接口，ASPI for SCSI设备接口
Web API；例如：Google地图API，新浪微博API，阿里云API市场




为什么要使用API

快速扩展功能
避免“造轮子”，提高开发效率
降低模块之间的耦合度




Web API协议与HTTP请求

一般采用HTTP作为底层协议


HTTP请求机制如下：


客户端向服务器发送一个请求
服务器给客户端一个响应，告诉客户端是否可以完成他的请求
请求 <====> 响应 机制


HTTP请求包含的内容：


为了构造有效的请求，客户端需要包含的4个部分：
·URL（API调用地址）
·请求方式
GET：请求服务器获取一个资源
POST：请求服务器创建一个新的资源
PUT：请求服务器编辑或更新一个已存在的资源
DELETE：请求服务器删除一个资源
·Headers（请求头）
提供了请求的元信息，是一个简单的项目列表；其中有客户端  发生请求的时间和请求主体大小、身份认证等信息
·Body（请求主体）
包含了客户端希望发送给服务器的数据


状态码

当成功调用API后，输了数据返回外，还会包含一个状态码

处理成功返回2XX；


服务端错误码是5XX，表示服务不可以（此时一般建议重试或联系商品页面的API服务商）
